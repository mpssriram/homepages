The Next Build

A forward-looking design audit — the visual and experiential changes to make next, separate from the code cleanup already in motion. The identity is strong; the job now is to make content lead and tighten the system underneath it.

Focus  Type · Color · Hierarchy · Signature moments Moves  11 Phases  3

The one-line thesis

You built a cinematic shell. Now build the system that lives inside it — and let the work, not the effects, do the talking.

Right now the cockpit hero earns full attention, then every section after it competes at the same volume: glow, tilt, scanlines, HUD labels, blinking cursors on everything. Confident design has a loud opening and a calm body. The moves below pull the volume down everywhere except where it counts, fix the broken handoff, and give the page a real type and color system to stand on.

01

Typography System

Highest leverage
Phase 1 Type ramp · Legibility

Set a real type ramp — and raise the floor

The current page runs body copy at ~12–13px, micro-labels at ~9px, much of it at 50–58% opacity. That's the single biggest thing holding the design at "above average." Commit to the ramp below: a confident display, a comfortable reading size, and a label that's small but never invisible. Same two fonts you already use.

02

Color & Tokens

Foundational
Phase 1 Tokens · Consistency

One red, named tokens, no more magic rgba

Three different reds are doing the same job across the codebase — the theme's #d81c3d, a hot rgba(255,45,85), and a flat #ef4444 — scattered as literals inside utility strings. Pick one, name the whole accent set, and reference tokens instead of raw values. This is what makes the cyber palette read as deliberate rather than improvised.

03

The Hero Handoff

Signature moment
Phase 1 Transition · Continuity

Dissolve the cockpit into the field — kill the blackout

The most visible flaw on the page. The warm, photoreal cockpit drops onto a cool abstract network field through a near-black curtain (#about::before fades to rgba(3,13,21,0.88)). It reads as a cut, not a transition. Three fixes, in order: soften that band to ~0.4 so the layers cross-fade; ease the hero's saturate/brightness down at the end of scroll so it cools into the field; and let the network nodes begin faintly inside the bottom of the hero so the eye reads one continuous space.

Now

Photoreal scene → black curtain → abstract field. A hard cut between two unrelated looks.

Next

Cockpit cools + dissolves into the network that's already faintly present. One space, settling.

Phase 2 Background density

Quiet the first screen after the hero

You currently go from a max-busy hero to a still-busy background (network + grid + horizon + sweep + scanlines + glows all at once). Drop the background density in the first viewport after the handoff so it feels like the scene settles, then let it breathe back in further down. Calm directly under the loudest moment is what makes the loud moment land.

04

Restraint & Card System

Re-usable
Phase 2 Hierarchy · Effects

Build a card hierarchy instead of one HUD treatment everywhere

Every card — value cards, lanes, projects, events, team — wears the same costume: glow border, tilt, scanline overlay, /// bar, UNIT 01 tag, blinking cursor, chip dots. When everything shimmers, nothing leads. Define three tiers: a feature card (one per section, keeps the full treatment), a standard card (flat panel, one accent line, real content), and a list row (text-led, minimal). Reserve motion and glow for the feature tier.

Now

~20 cards, all max-decoration, all equal weight.

Next

3 tiers. The eye knows where to go on every screen.

Phase 2 Content honesty

Cut the decorative "data slop"

The /// prefixes, BUILD_LOG / UNIT 02 tags, status pills and chip-dot rows are atmosphere standing in for information — they read as filler. Keep the device on the few places it means something (a real status, a real count) and strip it everywhere it's pure decoration. Less HUD makes the remaining HUD feel earned.

05

Proof & Imagery

Perceived quality
Phase 2 Imagery · Trust

Show the work and the faces

A builder community's strongest design asset is evidence it's real. Team avatars are generated initials; projects are described but never shown. Add a proof layer: real member photos, and project cards that show an actual screenshot/thumbnail of the thing. One screen of real work outperforms any new effect for perceived quality — and it's the natural home for the cyan/red framing you already love.

Phase 3 Imagery placeholders

Design real image slots now, even before content exists

Don't leave gaps — design intentional placeholder slots (labelled, framed in the brand HUD style) so the layout is built for imagery from day one and content can drop straight in. Avoids a redesign later when the photos arrive.

06

Motion & Spacing

Polish
Phase 3 Motion language

One entrance choreography, fewer infinite loops

Reveal-on-scroll is good; the always-looping pulses (glowing dots, breathing headings, animated borders) add ambient noise. Define a single entrance curve for content, keep one or two intentional ambient motions (the nav dot is enough), and let stillness be the default state.

Phase 3 Vertical rhythm

Lock a spacing scale and section rhythm

Spacing is currently a mix of rem literals and clamp() one-offs per page. Commit to one scale and apply it everywhere — consistent section padding, consistent gaps, consistent heading-to-body distance. Quiet structure the reader feels without noticing.
