Performance & Payload

4 findings
Critical P-01 Dead CSS · Bundle

~4,000 lines of legacy.css ship to every visitor, ~95% unused

src/legacy.css is imported in main.tsx and bundled in full, but the pages were migrated to inline Tailwind + cinematic-background.css. Selectors like .community-page, .team-hero, .journey-step, .culture-pill, .signal-strip appear nowhere in any .tsx file — only inside legacy.css itself.

Only a small hero subset stays live (.cockpit-hero, .static-hero, .sticky-viewport, .hero-canvas, .canvas-loader) — and several of those are also redefined in cinematic-background.css, so they're duplicated.

Fix

Delete the dead rules; keep only the live hero classes (ideally fold them into cinematic-background.css and drop the file). Easily the biggest CSS-payload win available.

High P-02 Assets · Load time

240-frame WebP scroll sequence on the homepage hero

public/cockpit-sequence/ holds 240 WebP frames driven by scroll (COCKPIT_FRAME_COUNT = 240). The mitigations are genuinely good — static poster fallback for mobile + reduced-motion, batched preloading, DPR capped at 1.5 — but on desktop this is still a large progressive download before the effect is smooth.

Fix

Trim to ~90–120 frames (scroll mapping won't notice), or switch to a compressed <video>/WebM with the canvas as fallback. Measure total transferred bytes and set a budget.

Medium P-03 Repo hygiene

302 source JPG frames committed in finalphotos/

The finalphotos/ezgif-frame-*.jpg set looks like the source input that the prepare:cockpit step turns into the WebP sequence. These are build inputs, not runtime assets — they bloat clone size and the repo with no benefit to the shipped site.

Fix

Move them out of the repo (or .gitignore them) and keep the WebP output as the only tracked sequence.

Low P-04 Dead code

No-op motion transform in CockpitHero.tsx

overlayOpacity = useTransform(scrollYProgress, [0, 1], [1, 1]) maps every input to a constant 1 — it does nothing and can be removed along with the prop it feeds into HeroOverlay.

Fix

Drop the transform and the overlayOpacity plumbing, or give it a real range if a fade was intended.

02

Architecture & DRY

3 findings
High A-01 Duplication

The same CTA button is copy-pasted as a giant class string 12+ times

The red-gradient call-to-action (inline-flex … border … bg-[linear-gradient(135deg,#ff3b3b,#ff4fd8)] …) is repeated verbatim across CommunityPage, ProjectsPage, EventsPage and TeamPage. This directly contradicts your own AGENTS.md: "Reuse existing UI components before adding new one-off patterns."

A side effect: the baked-in mt-8 margin lives inside the button string, so spacing is coupled to the component everywhere it's used.

Fix

Extract a <CtaLink variant="primary|ghost|cyan"> in components/ui/. One source of truth for padding, color and hover; pages just pass href + label.

Medium A-02 Duplication

SectionHeader and AnimatedBlock are redefined in four files

Near-identical SectionHeader and AnimatedBlock helpers are declared independently inside Community, Projects, Events and Team pages — and HomePage's RevealBlock is the same idea again. Five copies of two patterns drift apart over time.

Fix

Hoist both into components/ui/ and import them. One Reveal wrapper and one SectionHeader for the whole site.

Medium A-03 Design tokens

Three different "brand reds" used interchangeably

The theme defines --color-accent-red: #d81c3d, but pages bypass it with raw rgba(255,45,85,…) and #ef4444 in inline class strings. That's three reds doing the same job, scattered as magic values across hundreds of utility classes — hard to keep consistent or re-theme.

Fix

Pick one canonical red, expose cyan/red/pink as theme tokens, and reference accent-red / accent-cyan instead of literal rgba in markup.

03

Accessibility

3 findings
High AX-01 Legibility · Contrast

Micro-labels at ~9px and low-opacity body text likely fail WCAG AA

Status labels and HUD captions render at 0.56rem (~9px) and eyebrows at 0.62rem (~10px), often at text opacity 0.5–0.58 over a dark gradient. The micro-labels are below comfortable reading size, and the faint dim text probably misses the 4.5:1 contrast bar.

Fix

Floor body/caption text around 12–13px, raise the dimmest opacities, and spot-check contrast on the hero and card backgrounds.

Medium AX-02 Keyboard · Focus

No focus management on client-side route change

The custom router in App.tsx updates document.title and scroll position, but focus is never moved to the new page's heading and there's no skip-link. Keyboard and screen-reader users stay parked on the old, now-unmounted context after navigating.

Fix

On route change, move focus to the page's <h1> (or a visually-hidden route announcer) and add a "skip to content" link.

Low AX-03 Media

Team avatars are initials-only placeholders

The team grid renders generated initials rather than real photos, so there's no meaningful imagery or alt content for who's on the crew.

Fix

Drop in real member photos with descriptive alt text once content is ready (ties into C-01).

04

Content & Data

1 finding
High C-01 Launch blocker

Placeholder content throughout — not launch-ready

Team members are generic roles ("Club Coordinator", "Technical Lead", "Events Lead") rather than real people; the contact is a placeholder hello@devcell.club; the footer GitHub link points at github.com root; and events are templated ("Friday / Week 1", "Saturday / Week 1"). The structure is solid, but every concrete detail is stubbed.

Fix

Replace with real names, photos, a working contact, the actual club GitHub/socials, and a real upcoming-events feed before going public.

05

SEO & Metadata

3 findings
Medium S-01 SEO · Social

Only document.title changes per route — no per-page meta

The router updates the tab title but there's no per-route meta description, canonical URL, or Open Graph / Twitter card. As an SPA served from a single index.html, shared links and crawler snapshots get one generic description for every page.

Fix

Manage head tags per route (a small head-effect hook or react-helmet), and consider static pre-rendering for the five known routes.

Low S-02 Metadata

index.html missing favicon, touch icon and OG image

The document has a good theme-color and description, but no <link rel="icon">, no apple-touch-icon, and no og:image — so browser tabs and social previews fall back to defaults.

Fix

Add a favicon set, an apple-touch-icon, and a 1200×630 OG image referencing the cockpit hero.

Low S-03 Routing

/about and /community render the same page

Both routes map to CommunityPage. That's a deliberate alias, but two URLs serving identical content reads as duplicate content to crawlers and splits any inbound links.

Fix

Pick one canonical path and 301/redirect or rel=canonical the other.

06

Tooling & Robustness

2 findings
Medium T-01 DX · CI

No linting, formatting, or tests configured

package.json only has dev, build, typecheck and preview — no ESLint, no Prettier, no test runner. For a club codebase that new student contributors will touch (a stated goal of the site itself), that's a meaningful guardrail gap.

Fix

Add ESLint + Prettier with a shared config and a minimal CI check on PRs, so first-time contributors get instant feedback.

Low T-02 Scripts

Dangling prepare:cockpit script reference

package.json defines prepare:cockpit → node scripts/prepare-cockpit-sequence.mjs, but there's no scripts/ directory in the tree. Anyone running it (or regenerating the sequence) hits a missing-file error.

Fix

Commit the script (it's the bridge from finalphotos/ → WebP), or remove the npm entry if the pipeline lives elsewhere.

Recommended order

Where to start

  1. Strip legacy.css (P-01). Pure deletion, instant payload win, zero visual change — verify with a quick screenshot pass.
  2. Extract the CTA + shared SectionHeader/Reveal (A-01, A-02). Makes every later edit cheaper and enforces consistency.
  3. Lighten the hero sequence (P-02) and de-repo the source frames (P-03). Real first-load and clone-size improvements.
  4. Fix legibility & focus on route change (AX-01, AX-02). Low effort, broad reach.
  5. Swap in real content, then meta/SEO (C-01, S-01). The launch checklist.
